# **CallMeMaybe: Function Calling & LLM Evaluation Engine**

Welcome to **CallMeMaybe**, an advanced, state-driven Python engine designed for testing, evaluating, and managing Large Language Model (LLM) function calling capabilities.

This project utilizes a modular architecture (Engines, Layers, Models, States, and Tools) and relies on a local SDK (llm\_sdk) for core model interactions.

To make setup and execution as smooth as possible, this project uses a **Makefile** to orchestrate commands, relying **exclusively on [uv](https://www.google.com/search?q=https://docs.astral.sh/uv/)** under the hood for lightning-fast, highly isolated Python dependency and environment management.

## **⚡ Prerequisites**

To run this project, you need exactly two things installed on your system:

### **1\. uv (Python Package Manager)**

This project relies **exclusively** on uv. You do not need to pre-install Python; uv will handle it automatically based on the .python-version file.

* **macOS / Linux:** curl \-LsSf https://astral.sh/uv/install.sh | sh  
* **Windows:** powershell \-ExecutionPolicy ByPass \-c "irm https://astral.sh/uv/install.ps1 | iex"

*(Restart your terminal and verify with uv \--version)*

### **2\. make (Build Automation Tool)**

We use make to simplify terminal commands.

* **macOS:** Pre-installed (or install via xcode-select \--install).  
* **Linux:** Pre-installed on most distros (or sudo apt install make).  
* **Windows:** Recommended to use WSL (Windows Subsystem for Linux), Git Bash, or install via Chocolatey (choco install make).

## **🚀 Step 1: Clone or Download the Repository**

Navigate to your desired folder and grab the project:

cd /path/to/your/projects/folder  
git clone \<repository\_url\> CallMeMaybe  
cd CallMeMaybe

Ensure your terminal's current working directory is the root of the CallMeMaybe project (where the Makefile and llm\_sdk.zip are located).

## **⚠️ Step 2: Unzip the Local SDK (CRITICAL)**

The project relies on a local, proprietary package called llm\_sdk. Currently, this SDK is compressed inside the llm\_sdk.zip file. **The project will fail to build or run if this file is not extracted properly.**

You must extract it directly into the project root directory *before* running any installation commands.

### **Option A: Unzipping via Command Line (Recommended)**

**On macOS / Linux:**

unzip llm\_sdk.zip

**On Windows (PowerShell):**

Expand-Archive \-Path "llm\_sdk.zip" \-DestinationPath "." \-Force

### **Option B: Unzipping via File Explorer (GUI)**

1. Open your file explorer and navigate to the CallMeMaybe folder.  
2. Locate the file named llm\_sdk.zip.  
3. **Right-click** the file and select **"Extract All..."** (Windows) or **double-click** it (macOS).  
4. Ensure it extracts directly into the project root.

**Verification Check:**

After unzipping, you should see a new folder named llm\_sdk in your root directory sitting right next to pyproject.toml.

## **🛠️ Step 3: Project Setup & Installation**

Now that your llm\_sdk is correctly extracted, use the Makefile to set up the environment.

Run the following command from the root directory:

make install

**What this does under the hood:**

This triggers uv sync. uv reads the .python-version file, automatically fetches the exact Python interpreter required, reads the dependencies in pyproject.toml, detects your newly unzipped llm\_sdk, and creates an isolated, invisible .venv environment seamlessly.

## **🎯 Step 4: Running the Engine**

With the environment fully installed, you are ready to execute the main script. Because uv handles the virtual environment automatically, there is no need to manually activate anything.

Run the following command:

make run

*(Alternatively, simply running make or make all will trigger the same command).*

**What this does under the hood:**

This triggers uv run python \-Bm src. It ensures the command runs inside the project's isolated .venv, launches the \_\_main\_\_.py entry point, and executes the core engine.

The Makefile comes with built-in commands to maintain code quality and debug issues.

If you want to clear out generated results, cache folders (\_\_pycache\_\_), and compiled python files (\*.pyc):

## **📊 Understanding Inputs and Outputs**

The project relies on JSON files for its input state and output logging.

### **📥 Inputs (data/input/)**

* functions\_definition.json: Defines the schema of the tools/functions available to the LLM.  
* function\_calling\_tests.json: Contains the prompts and expected behaviors you are testing against the engine.

### **📤 Outputs (output/)**

* function\_calling\_results.json: After running make run, check this file. It contains the decoded outputs, state transitions, and evaluation results of the function calling tests. (Note: Running make clean will delete this folder).

## **🚑 Troubleshooting (Common Issues)**

**1\. "ModuleNotFoundError: No module named 'llm\_sdk'"**

* **Cause:** You skipped **Step 2**. The llm\_sdk.zip was not extracted, or it was extracted into a nested folder.  
* **Fix:** Delete any incorrect folders, extract llm\_sdk.zip directly into the root so that CallMeMaybe/llm\_sdk/pyproject.toml exists, and run make install again.

**2\. "make: command not found"**

* **Cause:** You are likely on Windows and do not have make installed.  
* **Fix:** You can either install make (via Git Bash or Chocolatey), or manually run the uv commands directly:  
  * Setup: uv sync  
  * Run: uv run python \-Bm src

**3\. "uv command not found"**

* **Cause:** Your terminal hasn't registered the uv installation path.  
* **Fix:** Close your terminal window completely and open a new one to refresh your PATH variables.
